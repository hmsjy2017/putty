summary: PSFTP failures don't always produce failure exit code

When running PSFTP with -batch, if a single command fails (e.g. because the target filesystem is full) but the SSH server returns an exit-status of zero then PSFTP itself returns an exit status of zero. In similar circumstances (and using the SFTP backend) PSCP returns failure.

PSFTP in batch mode already exits early if any command fails, so under those circumstances it would seem sensible for it to also return failure.
