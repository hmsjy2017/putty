summary: Key timeout in Pageant
class: wish: This is a request for an enhancement.
difficulty: tricky: Needs many tuits.
priority: medium: This should be fixed one day.

It would be nice if Pageant could be configured to automatically time out SSH keys it's holding, after a certain length of time (or perhaps a certain length of time in which they're unused). Then the keys would be deleteed, until reloaded.

If pageant-deferred-decrypt were implemented first, then a timed-out key could be put back into the 'ask for passphrase on first use' state, rather than being unloaded completely so that Pageant stopped advertising it.
