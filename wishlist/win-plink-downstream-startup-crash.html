summary: Windows Plink crashes on startup if it's a connection-sharing downstream
class: bug: This is clearly an actual problem we want fixed.

If Windows Plink tries to use an SSH connection that another PuTTY or Plink process has already set up, then it crashes with an illegal memory access at the point when it tries to print the message "Reusing a shared connection to this server".
