In Markus Kuhn's wcwidth.c, which PuTTY uses, there is a variant of wcwidth() that treats certain characters of ambiguous width as being double-width, rather than single-width as the default version does.

Currently the variant version is commented out and not used. We've had a few requests to add an option to use this variant, though.
