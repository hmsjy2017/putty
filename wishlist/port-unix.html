summary: Port to Unix
class: wish: This is a request for an enhancement.
difficulty: tricky: Needs many tuits.
priority: high: This should be fixed in the next release.
fixed-in: 0.54

It's currently being worked on. If you look at the nightly source snapshots, you should find a unix subdirectory, which should build you a Unix port of Plink, and also pterm - an xterm-type program which supports the same terminal emulation as PuTTY.

You can also build a Unix port of PuTTY itself. This isn't as useful a thing on Unix as it is on Windows; its major value is as a pathfinding effort for other ports. If anyone really has a use for it, we'd be interested to know why :-)

Ports of psftp, pscp, and puttygen are also now available.

We do not yet have a Unix port of Pageant (although another agent can be used).

See also the FAQ.

Update: as of 0.54, we think this is good enough to release.
