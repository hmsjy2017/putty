summary: Support for an "ssh" URL scheme
class: wish: This is a request for an enhancement.
difficulty: tricky: Needs many tuits.
priority: medium: This should be fixed one day.

Some people would like PuTTY to be able to accept "ssh" (and "rlogin", and "raw") URLs on the command-line, in the same way as it accepts "telnet" ones.

There is a provisional URI scheme registration for "ssh", but it's not particularly well-defined. PuTTY could usefully support at least the straightforward parts of it.

In the meantime, this workaround from Microsoft's site can be adapted to work with PuTTY.
