summary: Default bold text display accidentally changed, rendering 'bold black' invisible
class: bug: This is clearly an actual problem we want fixed.
difficulty: fun: Just needs tuits, and not many of them.
priority: high: This should be fixed in the next release.
absent-in: r9557 2012-06-09
present-in: r9559 2012-06-10 0.63
fixed-in: r10008 c0b6e0b9626baecab20ba6946dc26a75f187e2cf 2013-08-08 (0.64)

In 0.63, the default behaviour for bold text was inadvertently changed from changing the colour to changing the font. This change takes effect only when not using a session saved with a previous version of PuTTY.

It can easily be changed back by selecting 'Indicate bolded text by changing: The colour' on the Colours panel. Any sessions saved with 0.63 may need this adjustment.

Some server applications rely on 'bold black' text being distinguishable from a black background; so one consequence of this bug is that such text is rendered entirely invisible.
