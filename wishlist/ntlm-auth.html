summary: Ability to do NTLM authentication in various contexts (eg. proxy)
class: wish: This is a request for an enhancement.
difficulty: taxing: Needs external things we don't have (standards, users etc)

Occasionally people ask for support for NTLM authentication in various contexts (e.g. for HTTP proxy authentication).

NTLM authentication appears to be proprietary to Microsoft; what public documentation there is appears to be reverse engineered.
