In PuTTY 0.75, reconfiguring the RGB values of terminal colours mid-session (from the Change Settings dialog box) doesn't have any immediate effect, unlike previous versions.

(The reconfigured colours would belatedly take effect if the palette was subsequently reset, for instance by the escape sequence ESC ] R.)

After this fix, configured colour changes will take effect immediately, unless the palette has been overridden by an escape sequence from the server like ESC ] P nrrggbb; any server overrides will remain in effect until the palette is reset. This is a deliberate change from pre-0.75 behaviour.
