summary: Numeric keypad always uses English key bindings (e.g., decimal point) on Windows
class: bug: This is clearly an actual problem we want fixed.
difficulty: fun: Just needs tuits, and not many of them.
priority: high: This should be fixed in the next release.
absent-in: 0.70
present-in: 41e1a586fb956539a74bc446984a100e0138cd77 0.71
fixed-in: dfc215d0c07ebfc990304a3f6544420eb717e8ef (0.72)

The 0.71 Windows builds introduced a bug where keys typed on the numeric keypad, when in a mode where it should produce numeric characters, always produced the English character for the key rather than the appropriate locale-dependent character.

Notably, the decimal separator key always produced decimal point (.) even in locales where it would normally produce a decimal comma (,).
