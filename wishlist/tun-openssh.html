summary: OpenSSH-compatible tun VPN support
class: wish: This is a request for an enhancement.
difficulty: taxing: Needs external things we don't have (standards, users etc)
priority: never: We don't ever intend to fix this.

OpenSSH has a crude VPN system based on sending raw IP packets over an SSH channel. We're unlikely to implement this in PuTTY because:

  • The protocol appears to have no documentation other than the OpenSSH source code.
  • From what we've been able to work out of the protocol from the sources, it's ugly and depends on the boundaries between network packets being represented by boundaries between SSH_MSG_CHANNEL_DATA messages.
