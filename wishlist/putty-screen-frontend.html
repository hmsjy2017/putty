summary: screen(1) front-end
class: wish: This is a request for an enhancement.
difficulty: tricky: Needs many tuits.
depends: scripting
priority: low: We aren't sure whether to fix this or not.

We occasionally get requests to make PuTTY usable as a front-end to the screen terminal multiplexer, providing out-of-band control so that key sequences like C-a are freed up for use.

While this is clearly desirable, we want to avoid encoding knowledge of server-side applications into the core of PuTTY. This is a clear application of scripting and is unlikely to happen before that is implemented.
