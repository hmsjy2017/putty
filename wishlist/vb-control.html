summary: SSH client packaged as a Visual Basic component
class: wish: This is a request for an enhancement.
difficulty: taxing: Needs external things we don't have (standards, users etc)
depends: dll-frontend
priority: low: We aren't sure whether to fix this or not.

A lot of people ask for a Visual Basic component that implements an
SSH client, so that they can include it in their VB programs and
automatically get securely in touch with other systems and issue
commands.

I think the simplest way to do this would be to implement it as a
thin veneer on top of the DLL front end (once we've done _that_).

The major direct difficulty in this work is that I know nothing
about VB components and how to create one, and so I can't even think
about beginning this without some outside help. But also, it can't
be done unless the DLL work is done first.
Audit trail for this wish.
