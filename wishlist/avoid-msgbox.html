summary: Avoid using message boxes for "connection closed" and friends
class: wish: This is a request for an enhancement.
difficulty: tricky: Needs many tuits.
priority: medium: This should be fixed one day.

Perhaps, for people who don't like dialog boxes, messages like "connection closed" and fatal errors should be configurable to appear inside the window instead of popping up sudden message boxes. (For instance, if a network connection dies, apparently the message boxes get in the way of WinXP's "close group" feature.)

If we do this, we should probably include a means of distinguishing that sort of error output from anything the remote site can possibly send. This is potentially a hard problem, so it requires some design work. (See also discussion in gui-auth.)

Update, March 2019: vuln-auth-prompt-spoofing has forced us to do the design work. We can re-use the 'trust sigils' concept here.
